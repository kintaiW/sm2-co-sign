📋 概述

SM2 协同签名系统是一个基于国密 SM2 椭圆曲线算法的协同签名解决方案。系统采用客户端-服务端分离架构，通过密钥分片技术实现私钥的安全存储和协同签名。

核心原理：将完整私钥分为 D1（客户端持有）和 D2（服务端持有）两部分，签名时双方协同计算，确保完整私钥永不出现。

技术栈

Go 1.24 Rust 2021 SM2/SM3/SM4 SQLite Fiber v2 libsm

✨ 特性

🔐 密钥分片

私钥分为两部分独立存储，完整私钥永不出现，从根本上保障密钥安全。

✍️ 协同签名

客户端与服务端协同完成签名计算，支持标准 SM2 签名验证。

🔓 协同解密

支持 SM2 密文的协同解密，保障加密通信安全。

🌐 REST API

提供完整的 REST API 接口，支持业务系统快速集成。

🖥️ CLI 管理工具

提供用户管理、密钥管理、审计日志等 CLI 管理功能。

📦 多平台支持

客户端支持 Windows/Linux 动态库，便于集成到各类应用。

🚀 快速开始

克隆仓库

git clone --recursive https://github.com/kintaiW/sm2-co-sign.git
cd sm2-co-sign

启动服务端

cd backend
go mod tidy
go run cmd/server/main.go

构建客户端

cd client
cargo build --release

使用 Admin CLI

# 查看用户列表（管理端口: 7093）
curl http://localhost:7093/mapi/users

# 查看密钥列表
curl http://localhost:7093/mapi/keys

# 查看审计日志
curl http://localhost:7093/mapi/logs
⚠️ 安全提示：生产环境请务必修改默认管理员密码，并配置 HTTPS。

🛡️ 安全机制

密钥分片原理

客户端 服务端 | | |-- 1. 生成 d1 (随机数) | |-- 2. 计算 P1 = d1 * G | | | |--- POST /api/register {p1} ------->| | |-- 3. 生成 d2 (随机数) | |-- 4. 计算 Pa = d2Inv * P1 + (n-1) * G |<--- {userId, p2, pa} --------------| | | |-- 5. 存储 d1 本地 | 完整私钥: d = d1 * d2 - 1 (不存储)

安全特性

🔑 密钥隔离

客户端持有 d1，服务端持有 d2，完整私钥 d 不在任何一方存储。

🔒 传输安全

所有敏感数据使用 Base64 编码，生产环境必须使用 HTTPS。

🎫 Token 认证

使用 Bearer Token 认证，Token 有过期时间。

📝 审计日志

所有关键操作记录审计日志，支持追溯。
